# Proposal Generator デザインシステム

> **ドキュメントバージョン**: 1.0
> **最終更新日**: 2025-12-13

---

## 1. デザイン哲学 (Design Philosophy)

- **Less is more / Simple is better:** UIの装飾は最小限に留め、案件情報の閲覧・選択に集中できることを最優先とする
- **情報密度の最適化:** 案件一覧は一目で比較できるよう、必要十分な情報を適切な密度で表示
- **ライトモードデフォルト:** 日中の作業を想定し、ライトモードをデフォルトとする

---

## 2. レイアウトとスタイル (Layout & Style)

### 2.1. 基準グリッド

- **8pxグリッド**をデザインの基本単位とする
- スペーシング、コンポーネントサイズ、角丸などは原則として8の倍数（必要に応じて4px）で設計し、視覚的な一貫性を保つ

### 2.2. 境界線と影

- 要素の境界は、**影 (drop-shadow) を使用せず**、**細い単色のボーダー (border)** で表現する
- これにより、クリーンでフラットな印象を維持する

### 2.3. ページレイアウト

- **サイドバー + メインコンテンツ** の2カラム構成
- サイドバー: フィルター操作（固定幅 256px）
- メインコンテンツ: 案件一覧（フレキシブル幅）

---

## 3. カラー (Colors)

### 3.1. ニュートラルカラー

- ライトモードとダークモードの両方で利用できるグレーパレット
- shadcn/uiのデフォルトカラーを使用

### 3.2. セマンティックカラー

| 状態 | 用途 |
|------|------|
| **Success** | 成功状態（例: 取得完了） |
| **Warning** | 警告状態（例: 残り日数が少ない） |
| **Error** | エラー状態（例: API接続エラー） |
| **Info** | 情報状態（例: ヒント） |

具体的な色の値はshadcn/uiのデザイントークンに従う。

### 3.3. Badge カラー

| タイプ | 用途 | スタイル |
|-------|------|---------|
| タグ | 案件タグ（注目、限定公開等） | `secondary` |
| 特徴タグ | 案件特徴（経験者優遇等） | `outline` |

---

## 4. タイポグラフィ (Typography)

### 4.1. フォントファミリー

- **日本語・英語:** Geist Sans（Next.js デフォルト）
- **等幅:** Geist Mono

### 4.2. フォントスケール

- 8pxグリッドに準拠したフォントスケールを使用
- 具体的なサイズはshadcn/uiのデフォルトに従う

### 4.3. 主要テキストスタイル

| 要素 | クラス |
|------|--------|
| ページタイトル | `text-2xl font-bold` |
| セクションタイトル | `text-lg font-semibold` |
| カードタイトル | `font-semibold` |
| 本文 | `text-sm` |
| 補足テキスト | `text-sm text-muted-foreground` |

## 5. アイコン (Iconography)

- **アイコンライブラリ:** **Lucide (lucide-react)** を採用
- **スタイル:** シンプルでモダンな線画 (Line) スタイルで統一

### 使用アイコン一覧

| アイコン | 用途 |
|---------|------|
| `Clock` | 残り日数 |
| `Users` | 提案数 / 募集人数 |
| `Star` | クライアント評価 |
| `ExternalLink` | 外部リンク |
| `Loader2` | ローディング |

## 6. コンポーネントの状態 (Component States)

全てのインタラクティブな要素に対して、以下の状態を明確に定義する：

| 状態 | 説明 |
|------|------|
| **Default** | 通常状態 |
| **Hover** | マウスホバー時 |
| **Focus** | フォーカス時 |
| **Active** | アクティブ時（クリック中） |
| **Disabled** | 無効状態 |
| **Loading** | ローディング状態 |

具体的な視覚表現はshadcn/uiのデフォルトに従う。

## 8. 実装方針

### 8.1. 基本原則

1. **shadcn/uiの既存コンポーネントを確認**してから実装を開始する
2. **デフォルトのまま試す** → 微妙な場合は参考UIを決める → カスタマイズ
3. 細かい数値（px値）は指定せず、shadcn/uiのデフォルトを信頼する

### 8.2. カスタマイズが必要な場合

- 参考UIを決める（例: Linear, Notion, Vercel Dashboard, GitHub等）
- Tailwindクラスでの調整を優先
- 構造やプロップスの調整
- それでも要件に合わない場合のみ手動実装を検討

### 8.3. コンポーネント追加手順

```bash
cd frontend
npx shadcn@latest add <component-name>
```

## 9. レスポンシブデザイン

### 9.1. ブレークポイント

| ブレークポイント | 幅 | レイアウト |
|----------------|-----|----------|
| モバイル | < 1024px | 1カラム（フィルター上、一覧下） |
| デスクトップ | >= 1024px | 2カラム（サイドバー + メイン） |

### 9.2. Tailwind クラス

```tsx
// 2カラムレイアウト
<div className="flex flex-col lg:flex-row gap-6">
  <aside className="w-full lg:w-64">...</aside>
  <main className="flex-1">...</main>
</div>
```
