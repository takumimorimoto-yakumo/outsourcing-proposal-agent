## 1. 一般的な質問

### Q1.1. このツールは何ができますか？

**A**: クラウドソーシングサービス（ランサーズ、クラウドワークス）の案件ページから情報を取得し、AIを使って案件に最適化された提案文を自動生成するCLIツールです。

---

### Q1.2. どのサービスに対応していますか？

**A**: 現在、以下のサービスに対応しています：
- ランサーズ（`lancers.jp`）
- クラウドワークス（`crowdworks.jp`）

他のサービスは将来的に対応予定です。

---

### Q1.3. 無料で使えますか？

**A**: ツール自体は無料ですが、以下のサービスに依存しています：

| サービス | 料金 |
|----------|------|
| Gemini API | 無料枠あり（制限付き） |
| GitHub API | 無料（レート制限あり） |

---

### Q1.4. 生成される提案文の品質は？

**A**: AIが案件内容を分析し、以下を考慮した提案文を生成します：
- 案件の要件への理解・共感
- 具体的な提案内容
- GitHubの実績（設定時）
- 適切な文字数（1,500〜1,800文字）

ただし、生成後に内容を確認・修正することを推奨します。

## 2. 機能に関する質問

### Q2.1. GitHub連携は必須ですか？

**A**: いいえ、オプションです。GitHub連携なしでも提案文は生成できます。

```bash
# GitHub情報なしで生成
proposal-gen <URL> --no-github
```

GitHub連携を有効にすると、技術力・実績のセクションがより具体的になります。

---

### Q2.2. 複数のプロフィールを使い分けられますか？

**A**: はい。`profile.yaml` に複数のプロフィールを定義し、コマンドで切り替えられます。

```bash
# デフォルトプロフィール
proposal-gen <URL>

# チームプロフィール
proposal-gen <URL> --profile team
```

---

### Q2.3. 生成した提案文をファイルに保存できますか？

**A**: はい。`--output` オプションで保存先を指定できます。

```bash
# ファイルに保存
proposal-gen <URL> --output proposal.txt

# クリップボードにコピー
proposal-gen <URL> --copy
```

---

### Q2.4. 一度に複数の案件を処理できますか？

**A**: 現在のバージョン（v1.0）では対応していません。1回の実行で1つの案件を処理します。

将来のバージョンで一括処理機能を追加予定です。

---

### Q2.5. ログインが必要な案件にも対応していますか？

**A**: いいえ、現在は公開案件のみに対応しています。ログインが必要な案件へのアクセスは将来的に検討中です。

---

### Q2.6. 生成された提案文の履歴は保存されますか？

**A**: 現在のバージョンでは自動保存されません。`--output` オプションで手動保存してください。

## 3. 設定に関する質問

### Q3.1. 設定ファイルはどこに保存されますか？

**A**: 以下の順番で読み込まれます：

1. コマンドライン引数で指定したパス
2. カレントディレクトリの `config/`
3. ホームディレクトリの `~/.proposal-gen/`

```bash
# 設定ファイルの場所を確認
proposal-gen config show --path
```

---

### Q3.2. APIキーを安全に管理するには？

**A**: 以下の方法を推奨します：

1. **環境変数を使用**（最も安全）
   ```bash
   export GEMINI_API_KEY="your_api_key"
   ```

2. **設定ファイルのパーミッションを制限**
   ```bash
   chmod 600 ~/.proposal-gen/settings.yaml
   ```

3. **設定ファイルをバージョン管理に含めない**
   ```
   # .gitignore
   .env
   config/settings.yaml
   ```

---

### Q3.3. 設定をリセットするには？

**A**: 設定ディレクトリを削除して再初期化します。

```bash
# 設定を削除
rm -rf ~/.proposal-gen

# 再初期化
proposal-gen config init
```

---

### Q3.4. 自己紹介文をカスタマイズするには？

**A**: `profile.yaml` を編集します。

```bash
proposal-gen config edit --profile
```

`introduction.self_intro` セクションに自分の経歴・スキルを記載してください。

---

### Q3.5. AIの創造性を調整するには？

**A**: `settings.yaml` の `temperature` パラメータを調整します。

```yaml
gemini:
  temperature: 0.7  # 0.0〜1.0
```

- **低い値（0.3〜0.5）**: より保守的で一貫した出力
- **高い値（0.7〜0.9）**: より創造的で多様な出力

## 4. 技術的な質問

### Q4.1. どのAIモデルを使用していますか？

**A**: Google の Gemini API（gemini-pro モデル）を使用しています。

設定で変更可能：
```yaml
gemini:
  model: "gemini-pro"  # または "gemini-pro-vision" など
```

---

### Q4.2. スクレイピングはどのように行われますか？

**A**: Playwright を使用してブラウザを自動操作し、案件ページの情報を取得します。

特徴：
- JavaScript 実行後の DOM を取得
- 人間らしい振る舞い（待機時間、スクロール）を実装
- ヘッドレスモードで実行

---

### Q4.3. オフラインで使用できますか？

**A**: いいえ。以下のサービスへのインターネット接続が必要です：
- 案件サイト（ランサーズ/クラウドワークス）
- Gemini API
- GitHub API（オプション）

---

### Q4.4. プロキシを使用できますか？

**A**: 現在のバージョンでは直接サポートしていません。システムレベルでプロキシを設定してください。

```bash
export HTTP_PROXY=http://proxy:port
export HTTPS_PROXY=http://proxy:port
```

---

### Q4.5. Windows で動作しますか？

**A**: はい。Windows 10/11 で動作します。PowerShell または WSL2 での使用を推奨します。

---

### Q4.6. Docker で使用できますか？

**A**: 現在、公式 Docker イメージは提供していません。将来的に対応予定です。

ただし、Playwright を含む Docker 環境を自身で構築することは可能です。

## 5. 料金・制限に関する質問

### Q5.1. Gemini API の無料枠はどのくらい？

**A**: Google AI Studio の無料枠（2025年12月時点）：
- 15 RPM（1分あたりのリクエスト数）
- 1 million TPM（1分あたりのトークン数）
- 1,500 RPD（1日あたりのリクエスト数）

詳細は [Google AI Studio](https://makersuite.google.com/) で確認してください。

---

### Q5.2. GitHub API の制限は？

**A**:
| 認証 | 制限 |
|------|------|
| 認証なし | 60 requests/hour |
| トークン認証 | 5,000 requests/hour |

GitHub トークンを設定することで制限を緩和できます。

---

### Q5.3. 1日に何回くらい使用できますか？

**A**: Gemini API の無料枠では 1日約 1,500 回の生成が可能です。通常の使用では十分な量です。

---

### Q5.4. API 料金が発生する場合は？

**A**: 無料枠を超えた場合、Google Cloud の料金体系に従って課金されます。

課金を防ぐには：
1. Google AI Studio で使用量を監視
2. 月次の使用量上限を設定

---

### Q5.5. スクレイピングの回数制限はありますか？

**A**: ツール側での制限はありませんが、以下を守ってください：
- 連続リクエストは5秒以上間隔を空ける
- 短時間での大量アクセスを避ける
- 各サービスの利用規約を遵守する
